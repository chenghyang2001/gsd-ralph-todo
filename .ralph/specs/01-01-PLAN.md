<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell args -> todo.py | User/Ralph-supplied command and task text enter via argv (untrusted length/content, but local single-user) |
| tasks.json <-> todo.py | On-disk JSON store read/written each invocation; may be absent, empty, or hand-edited/corrupt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| ----------- | ---------- | ----------- | ------------- | ----------------- |
| T-01-01 | Denial of Service | load_tasks (missing tasks.json) | mitigate | Absent file returns [] and exits normally (ERR-03) — verified by test_list_missing_tasks_json |
| T-01-02 | Tampering | load_tasks (corrupt/hand-edited JSON) | mitigate | Wrap json.load in try/except JSONDecodeError -> print clear error to stderr, exit 1; never emit a raw Python traceback |
| T-01-03 | Tampering | TASKS_PATH resolution | mitigate | Path fixed to Path(**file**).resolve().parent / "tasks.json"; no user-controlled path component, so no path traversal via argv |
| T-01-04 | Information Disclosure | error messages | accept | Local single-user CLI, no secrets; error text may name the file path — acceptable for a local tool |
| T-01-05 | Elevation of Privilege | argument handling | accept | argparse rejects unknown commands; no shell/eval, no dynamic import of argv — no injection surface |
| T-01-SC | Tampering | supply chain (package installs) | accept | Zero third-party runtime deps (stdlib only). pytest is the sole dev/test dependency and is assumed present in the Ralph environment; no install step in this plan |
</threat_model>
