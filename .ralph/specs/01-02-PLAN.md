## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell args -> todo.py | id argument arrives via argv; parsed as int by argparse (non-int -> argparse error, exit 2) |
| tasks.json <-> todo.py | done/rm read-modify-write the on-disk store; store may lack the requested id or be absent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| ----------- | ---------- | ----------- | ------------- | ----------------- |
| T-02-01 | Denial of Service | cmd_done / cmd_rm on absent or empty store | mitigate | load_tasks returns [] when file absent; id-not-found path prints error and exits 1 rather than crashing (ERR-01) |
| T-02-02 | Tampering | non-integer id argument | mitigate | argparse id positional declared type=int; malformed id rejected by argparse before handler runs (no crash, clean usage error) |
| T-02-03 | Spoofing/Integrity | duplicate or missing id in hand-edited tasks.json | accept | done/rm act on the first match / all matches deterministically; malformed store is out of the normal single-user flow — acceptable for a local tool |
| T-02-04 | Information Disclosure | error message content | accept | Local single-user CLI, no secrets; echoing the offending id is acceptable and aids debugging |
| T-02-SC | Tampering | supply chain (package installs) | accept | No new dependencies; stdlib only. pytest remains the sole dev/test dependency, assumed present in the Ralph environment |
</threat_model>
